# 금융기관조회서 변환기 (AutoFinancialConfirmation)

은행이 보낸 **전자 금융기관조회서 PDF 묶음**을 자동으로 읽어, 감사용 표준 엑셀
(예금·대출·지급보증·담보 등)로 정리하고, 회사가 제시한 명세(PBC)와 대사까지 해주는
도구입니다. 설치형 프로그램(GUI)과 소스 양쪽으로 쓸 수 있습니다.

```
[입력]  금융기관조회서(전자)_[회사]_[날짜].zip   (PDF 여러 장)
   │   ① 표 인식으로 칸별 추출  ② 감사 계정분류  ③ 회사명세(PBC) 대사
   ▼
[출력]  용도별 엑셀
   · BANK / INSURANCE / GUARANTEE / INVESTMENT  : 섹션별 시트(1표=1시트)
        └ 각 파일 첫 시트 '0_요약' : 금융기관·통화별 합계를 엑셀 함수로 자동 집계
   · PBC대사 : 회사명세 ↔ 조회서 (자산·부채 / 차이·누락·미회수)
```

## 설계 원칙

- **결정적(deterministic) 추출 — 실행 중 AI 호출 0회.** 같은 PDF면 항상 같은 결과가
  나와 재현·검증이 가능합니다. (AI는 *이 프로그램을 만드는 단계*에서만 사용)
- **사람의 판단이 필요한 항목은 추측하지 않고 `검토필요`로 표시.** 자동화가 감사인의
  판단을 대체하지 않습니다.
- **바뀔 규칙은 코드 밖 설정(YAML)으로 분리.** 계정분류 기준·기관 양식·섹션 스펙을
  `configs/*.yaml` 로 동결해, 코드를 고치지 않고 규칙만 바꿀 수 있습니다.
- **표 인식(PyMuPDF find_tables).** 평면 텍스트가 아니라 표의 행·열 격자를 복원해
  추출하므로, 페이지 넘김 오염이 없고 '인출제한' 같은 칼럼까지 정확히 잡습니다.

## A. 설치형 프로그램으로 쓰기 (권장 — 비전문가용)

`build_exe.bat` 로 만든 **단일 실행파일** 하나면 됩니다(파이썬 설치 불필요).

1. `금융기관조회서변환기.exe` 더블클릭 → 창이 열림
2. ① 조회서 ZIP 선택 · ② 회사 예금명세(선택) · ③ 결과 폴더
3. **[변환 시작 ▶]** → 끝나면 결과 폴더 열림

> 처음 실행 시 Windows "PC 보호" 경고가 뜨면 → **추가 정보 → 실행** (서명 안 된 빌드).
> 빌드 방법은 [배포안내.md](배포안내.md) 참고.

## B. 소스에서 실행하기 (개발자용)

```powershell
# 준비 (최초 1회)
pip install -e .

# GUI 실행
python -m afc.gui

# 명령행 — 조회서 zip → 엑셀 (선택: 회사명세로 대사)
python -m afc.run "C:\...\금융기관조회서(전자)_[삼화전기]_[2025-12-31].zip"
python -m afc.run "C:\...\조회서.zip" --pbc "C:\...\회사제시_예금명세.xlsx"

# 테스트
python -m pytest tests/ -q
```

출력은 `output/<회사>/<타임스탬프>/` 에 보존됩니다(`_latest.txt` 가 최신 실행 표시).

| 산출 파일 | 내용 |
| --- | --- |
| `..._BANK_<회사>.xlsx` | `0_요약` + 예금·대출·지급보증·담보 (섹션별 시트) |
| `..._INSURANCE / GUARANTEE / INVESTMENT_<회사>.xlsx` | 보험·보증·증권 (해당 기관이 있을 때) |
| `..._PBC대사_<회사>.xlsx` | 대사·계정과목별·금융기관별·통화별·PBC대사(`--pbc` 시) |
| `00_검증리포트.md` | 취합 현황·기관 분류·수동 확인 필요 항목 |
| `confirmations.jsonl` | 조회서별 원시 레코드(원문 보존) |

## 핵심 기능

- **표 인식 추출**: 예금(§1)·대출(§2-2)·지급보증(§3)·담보(§5·§9) 등 섹션을 칸별로 추출.
  통화/금액 분리, 날짜 `YYYY-MM-DD`, **인출제한** 칼럼 포함. 은행별 양식 편차 흡수.
- **감사 계정분류**: 현금및현금성자산 / 단기·장기금융상품 / 사외적립자산(퇴직연금) /
  검토필요, 그리고 대출 → **단기·장기차입금(부채)**. 자산·부채를 섞지 않고 집계.
- **0_요약 (라이브 수식)**: 각 카테고리 파일 첫 시트에 금융기관 × 통화 합계를
  `SUMIFS`/`COUNTIFS` 로 자동 집계(데이터 수정 시 자동 갱신). 같은 워크북 내부 참조.
- **PBC 대사**: 회사 예금명세 ↔ 조회서를 (계좌번호+통화) 키로 양방향 대사 →
  금액 차이 · 회사명세 누락 · 조회서 미회수를 자동 표시.
- **상태(취합 현황) 판정**: `완료` / `거래하지 않는 금융기관` / `거래 없음(검토필요)`.

## 디렉토리

```text
afc/
  run.py                 end-to-end 진입점 (python -m afc.run) · run_pipeline()
  gui.py                 설치형 GUI (tkinter) · 명령행 모드 겸용
  paths.py               리소스 경로 (PyInstaller 번들/개발 양쪽 대응)
  tables.py              표 인식(find_tables) → 섹션 자동분류·페이지병합·잡음제거
  extract.py             헤더·상태·예금·대출(통화/금액·날짜·인출제한) 도출
  sections.py            cck 섹션·컬럼 스펙 로더 + PDF 섹션 분할
  reconciliation.py      금융상품·대출 → 감사 계정분류(자산/부채)
  pbc.py                 회사제시(PBC) 명세 로더 + 조회서↔PBC 양방향 대사
  output.py              다중 파일 출력(카테고리별 0_요약 라이브수식 + 섹션 시트)
  excel_writer.py        대사·계정과목별·통화별·PBC대사 시트 작성
  make_sample_pbc.py     (데모) 조회서 → 모의 PBC 생성 (차이/누락 시딩)
  schema.py              canonical 레코드 / 검증 객체
  pdf_inventory.py       zip/PDF 메타·섹션 인벤토리
  evaluation/diff_xlsx.py  cck 대비 셀 단위 비교(회귀용)
configs/
  institutions.yaml      기관 분류·섹션 시그니처·상태 문구·통화
  account_mapping.yaml   금융상품종류 → 감사 계정분류 규칙 (만기 단기/장기 기준)
  sections.yaml          4개 카테고리 섹션·컬럼 스펙 (cck에서 동결, 런타임 비의존)
tests/                   회귀 테스트 (합성 데이터, 60개)
build_exe.bat            PyInstaller 단일 exe 빌드
배포안내.md              빌드/배포 안내 (개발자·사용자용)
```

## 현재 커버리지

| 영역 | 상태 |
| --- | --- |
| 헤더·취합상태 판정 + 수동검토 플래그 | ✅ 전체 PDF |
| 예금(§1) · 대출(§2-2) | ✅ 칸별 추출 (통화/금액·날짜·인출제한·약정한도/잔액) |
| 지급보증(§3) · 담보(§5·§9) · 보험/증권 §1 | ✅ 표 인식 추출 (삼화전기 cck 행수 일치) |
| 대출 → 차입금(부채) 대사 편입 | ✅ 단기/장기차입금, 자산·부채 분리 집계 |
| PBC 대사 (양방향: 차이·누락·미회수) | ✅ 계좌·통화 키 매칭 (예금 기준) |
| 0_요약 라이브 수식 시트 | ✅ 카테고리 파일별 |
| 검증 데이터 | 삼화전기(15건·4카테고리, cck 일치) · 정밀회사(3건) |
| 남은 작업 | 사용제한 칼럼 대사 반영, §2-1 총한도·§4·§6~8 시트화, 외화 환율 자동, 합산검증 |

## ⚠️ 데이터·보안

- 실제 고객 조회서·회사 제시 자료(PBC)는 **비밀유지 대상**입니다. 저장소(git)에는
  코드만 두고, 실데이터·산출물·빌드물(`output/`, `feedback/`, `dist/`)은 제외합니다.
- 본 도구는 **시제품**이며, 자동 분류 결과는 반드시 감사인이 검증해야 합니다.
